# Admin 后台重构需求

## 背景

当前 admin 页面为单体文件 `frontend/pages/admin.js`（3165 行），所有面板在同一页面垂直堆叠，存在以下问题：

1. 首屏加载时并行拉取所有 API，即使管理员只想看一个面板
2. 3165 行单文件难以维护，面板间耦合低但物理混合
3. 移动端体验差：长页面滚动，无法快速定位目标功能
4. 无导航结构，依赖页面内滚动跳转

## 目标

- 将 admin 拆分为左侧导航 + 多页面结构
- 每个页面按需拉取数据，不加载无关 API
- 移动端可操作（侧边栏收起为汉堡菜单或底部 tab）
- 保持 `/admin` 为总入口，二级页面使用 `/admin/*` 嵌套路由
- 单一角色：超级管理员，不引入权限模型

## 约束

- 不引入细粒度权限模型
- 保留 `/admin` 作为总入口
- 移动端必须可操作（非只读）
- 不改变后端 API，前端路由拆分不增加后端工作量

## 验收标准

1. 原 `admin.js` 中所有功能均可在新路由结构下访问
2. 切换页面只拉取当前页面数据，不触发其他页面 API
3. PC 端左侧导航常驻，移动端侧边栏可收起
4. 移动端所有操作（触发刷新、查看日志、支付测试等）均可完成
5. 原有测试全部通过或已迁移
