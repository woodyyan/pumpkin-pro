# Requirement: 因子实验室 权重百分比输入与手动应用

## 背景

因子实验室页面的“因子与权重”当前使用 `0-1` 的小数权重（例如 `0.3` 表示 30%），不符合多数用户的输入习惯，且容易误填。

同时，右侧股票列表当前会随左侧编辑实时变化（自动应用），在用户调整多个权重时会造成列表抖动与不可控，影响使用效率。

## 目标

1. 将权重输入口径改为 `0-100` 的百分比，提升输入直觉与降低错误率。
2. 引入“草稿-已应用”分离的手动应用模式，让右侧列表更新可控且交互稳定。
3. 保留首屏可用性：进入页面即可看到默认结果。

## 功能需求

### 1. 百分比权重输入

1. 权重输入范围为 `0` 到 `100`（含边界），输入值按百分比语义解释。
2. 允许小数百分比，例如 `12.5%`，最多保留 2 位小数。
3. UI 展示必须带 `%`（例如输入框后缀或内联展示）。
4. 用户输入 `30` 表示 `30%`，不要求用户输入 `0.3`。

### 2. 首屏默认结果

1. 首屏仍然自动展示一版“默认 7 因子等权结果”。
2. 默认结果视为“已应用配置（Applied）”的初始值。

### 3. 手动应用（Draft vs Applied）

1. 左侧编辑区维护“草稿配置（Draft）”，允许用户连续修改而不影响右侧列表。
2. 右侧股票列表仅基于“上一次已应用配置（Applied）”进行计算与展示。
3. 左侧底部提供按钮 `启动计算`，点击后将 Draft 应用为 Applied 并触发右侧列表更新。
4. `启动计算` 按钮文案固定为“启动计算”，按钮旁不增加辅助说明。

### 4. 右侧列表交互规则

1. 用户改了权重但还没点“启动计算”时：
2. 右侧排序、翻页要继续可用。
3. 排序与翻页必须严格基于“上一次已应用的配置（Applied）”的结果，不得基于 Draft。

### 5. 重置

1. `重置` 按钮语义为“仅重置左侧草稿（Draft）”。
2. 重置不影响右侧当前 Applied 的展示结果。

## 非功能需求

1. 可理解性：用户需要能清晰感知“右侧展示的是上一次已应用结果”，但本次不通过新增辅助文案来解释。
2. 可维护性：Draft 与 Applied 状态必须结构清晰、边界明确，避免混用导致难以排查的 bug。
3. 兼容性：百分比输入转换为内部权重口径需保持前后端一致。

## 约束

1. 不更改 `启动计算` 按钮文案，不增加说明文字。
2. 右侧排序/分页行为在未应用 Draft 时继续可用，且只基于 Applied。
