# Requirement: 卧龙 AI 精选模拟组合指标改造

## 背景

行情看板里的“卧龙AI精选模拟组合”已经能展示累计收益、收益曲线和当前成分股，但当前信息结构还有 4 个明显问题：

1. 成分股层面只有股票名单，没有清楚告诉用户“按买入价算，现在这只股票是浮盈还是浮亏”。
2. 顶部指标仍保留“基准收益”“超额收益”，而用户更关心这套组合自己赚了多少、跑得稳不稳、用了多久。
3. 收益曲线和后端结果仍然保留 benchmark 计算链路，维护成本高，也与新的页面表达不一致。
4. 新增指标后，如果不重新排版，页面会变得拥挤，破坏现有“精简、轻量”的看板体验。

## 目标

在不引入盘中实时依赖的前提下，把模拟组合改造成“收盘后可解释、口径清楚、信息更完整”的简洁结果页。

## 需求范围

### 1. 当前成分股增加个股浮盈/浮亏

1. 每只成分股展示“较买入价”收益率。
2. 买入价口径：按该股票当前连续持有周期的首个纳入交易日收盘价。
3. 最新价口径：按当前最新 `source_trade_date` 对应的收盘价；若该交易日无价，可向前取该市场最近可用交易日收盘价。
4. 文案不写“今天涨跌”，避免把收盘后静态数据误解为盘中实时盈亏。
5. 行内同时展示买入价、最新价与收益率，减少歧义。

### 2. 顶部指标改造

保留：
- 累计收益

新增：
- 成立天数
- 起始日
- 昨日收益率
- 本月收益率
- 最大回撤
- 波动率
- 日胜率

删除：
- 基准收益
- 超额收益

### 3. 收益曲线改造

1. 收益图只保留模拟组合单曲线。
2. tooltip 可展示累计收益、单日收益、回撤。
3. benchmark 曲线从前端展示、后端结果和重建脚本里一起移除。
4. 后端不再保留 ranking portfolio 的 benchmark 收益计算链路。

### 4. 指标口径

1. 成立天数按自然日统计，并显式展示起始日。
2. 昨日收益率：取最新一个有收益记录的组合单日净收益率，包含调仓成本影响。
3. 本月收益率：按自然月边界计算；若成立未满一个自然月，则与累计收益率一致。
4. 最大回撤：以组合净值序列的历史峰值回落幅度统计，返回成立以来最深回撤。
5. 波动率：基于成立以来日收益率序列计算年化波动率，年化因子固定为 252。
6. 日胜率：成立以来正收益交易日数 / 全部有收益记录交易日数。

### 5. 交易日与跨市场要求

1. A 股和港股都要支持。
2. 用户展示日期统一使用 `source_trade_date`，不能用 `computed_at` 代替。
3. 涉及前一交易日、最新可用收盘价时，必须按市场口径处理，不能用自然日减一天推算。

## 非目标

1. 本次不接入盘中最新价，不做实时“今日浮盈”。
2. 不改动其他模块（如持仓归因、回测页）的 benchmark 口径。
3. 不新增第二条图表叠加曲线，避免页面复杂化。
