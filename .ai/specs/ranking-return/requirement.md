# Requirement: 卧龙 AI 精选排行榜涨幅口径

## 背景

行情看板的卧龙 AI 精选排行榜展示"连续上榜 N 日"与"上榜以来"涨幅。历史实现中，连续上榜天数按最新快照向前追溯连续日期，但涨幅按历史首次上榜快照计算。股票断档后重新上榜时，两个指标窗口不一致，容易被用户理解为当前连续周期出现异常涨幅。

## 目标

让排行榜主展示的涨幅反映当前连续上榜周期以来的表现，保证"连续上榜"与"上榜以来"两个指标口径一致。

## 功能需求

1. 后端 `RankingItem.return_pct` 按当前连续上榜周期计算。
2. 连续周期从最新排行榜快照日期向前逐日连续追溯；断档后重新上榜应重置起点。
3. 连续 1 天时，涨幅起点与终点均为最新快照，若价格可用则返回 0%。
4. 若连续周期起点或最新快照没有可用正价格，则保留现有容错：使用周期内最早/最新可用正价格；如果完全无有效价格则返回 `null`。
5. 历史首次上榜以来涨幅不再作为排行榜主指标。

## 数据修复需求

1. `quadrant_scores.exchange` 必须随计算结果更新，避免 688/689 等科创板股票长期保留旧的 `SZSE` 错误值。
2. `quadrant_ranking_snapshots.close_price` 已存在的正价格可能并非最终收盘价，需要通过独立回填命令刷新历史快照价格。
3. 业务 API 不在读取时自动改写历史快照，避免隐藏数据修复动作。
