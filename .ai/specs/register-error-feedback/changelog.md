# 注册错误提示优化

## 2026-06-05

- 注册接口不再把邮箱为空、邮箱格式错误、密码为空、密码过短统一折叠为 `invalid auth input`，而是返回稳定错误码：`EMAIL_REQUIRED`、`INVALID_EMAIL`、`PASSWORD_REQUIRED`、`PASSWORD_TOO_SHORT`、`EMAIL_EXISTS`。
- 注册弹窗新增前置引导：在输入密码时直接展示“至少 8 位”的硬性要求，并把“推荐同时包含字母和数字”作为建议而不是拦截条件。
- 当邮箱已存在时，错误提示会明确告诉用户该邮箱已注册，并直接提供“登录”与“找回密码”两个后续动作，减少注册链路中的误伤流失。
