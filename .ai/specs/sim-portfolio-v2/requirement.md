# Requirement: 模拟组合 v2 Calendar-driven Pipeline

## 背景

旧模拟组合链路依赖排行榜快照日期推断下一交易日，并通过 `quadrant_ranking_portfolio_market_prices` 兼作价格需求、价格缓存和补价目标。该设计在以下场景中不可维护：

1. 某交易日四象限或排行榜快照失败，日期会从已有快照序列中消失，系统无法显式识别缺口。
2. A 股和港股交易日历不同，休市日会被误判为缺少开盘价或收盘价。
3. Admin 的“补齐建仓开盘价 / 补齐收盘价 / 检查日期数据”等按钮只能修补局部字段，无法定位上游缺快照、缺成分股、缺价格需求或事实表未生成等根因。
4. 旧事实表和旧组合快照链路积累了较多技术债，继续缝补会扩大复杂度。

## 目标

建设新的 `Sim Portfolio v2`：以市场交易日历为唯一日期基准，采用严格模式生成正式收益口径，并提供 Admin pipeline 操作台追踪每个市场、每个交易日、每个阶段的状态。

## 功能需求

1. 新链路必须由市场交易日历驱动，支持 A 股 `ASHARE` 和港股 `HKEX` 独立判断交易日、上一交易日和下一交易日。
2. 严格模式：如果交易日应有信号、选股、开盘价、收盘价或事实表但任一前置阶段缺失，不生成该日正式收益，不沿用旧持仓伪造连续曲线。
3. 新增 pipeline 状态与运行记录，Admin 能看到阶段状态、阻断原因、缺口数量和建议动作。
4. 新增模拟组合专用信号批次、选股批次、价格需求和 v2 事实表，不迁移旧历史数据。
5. 重构完成后从最新完整交易日开始启动模拟组合，A 股和港股可独立水位推进。
6. Public `/portfolio-tracking` 继续提供现有接口形态，但底层读取 v2 已验证事实表；没有 verified 数据时展示初始化/等待状态。
7. 旧 Admin 模拟组合按钮下线，不再暴露“补齐建仓开盘价”“补齐收盘价”“检查该日期数据”“设置全局开始信号日”等旧操作。

## 非目标

1. 不迁移旧模拟组合历史数据。
2. 不修复旧 `quadrant_ranking_portfolio_*` 和旧 `portfolio_*` 模拟组合事实表中的历史问题。
3. 不用实时价进入正式净值曲线。
4. 不要求 A 股和港股使用同一个开始信号日。

## 约束

1. 交易日期不得从 `computed_at - 1 day`、自然日加减或下一条快照推断。
2. `entry_open` 与 `valuation_close` 必须精确匹配目标市场交易日，不允许向前取旧价格作为当日价格。
3. 休市日必须标记为 skipped，不允许报缺开盘价或缺收盘价。
4. 新表和新代码统一使用 `sim_portfolio_v2` / `pipeline` 命名，与旧链路隔离。
