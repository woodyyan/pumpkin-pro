# Requirement: source_trade_date 统一展示

## 背景

行情看板 3 个位置（风险机会全景图、卧龙 AI 精选排行榜、模拟组合当前成分股）的数据日期展示有误：凌晨 02:00 计算完成的数据显示为计算当天的日期，但实际基于前一天收盘数据。用户会误以为数据是当天收盘后的。

## 目标

让用户看到的日期反映数据所基于的真实收盘交易日，消除歧义。

## 功能需求

1. 后端：所有日级收盘后批处理写入 `source_trade_date`，按市场交易日历查询前一交易日。
2. 后端：相关 API 响应包含 `source_trade_date`。
3. 前端：统一 helper 将 `source_trade_date` 渲染为 `按 {date} 收盘后数据生成`。
4. 前端：禁止直接使用 `computed_at` 作为用户口径日期。
5. 全局规范：后续新增模块（行业榜单、策略评分、因子快照等）接入同一规则。

## 非功能需求

- A 股和港股交易日历各自独立维护。
- `source_trade_date` 作为批次级元数据，同一批结果共享。
- 历史回溯页和管理后台可同时展示 `source_trade_date` 和 `computed_at`。

## 约束

- 不能用 `computed_at - 1 day` 推算收盘日。
- 交易日历数据必须准确，否则所有依赖模块的日期都会错。
