# 个股详情页 UI 重构需求

## 背景

`/live-trading/[symbol]` 个股详情页承载行情、AI 分析、新闻、基础面、技术指标、持仓、交易信号等大量内容。旧版采用纵向长页面堆叠，页面过长，用户难以快速完成核心任务。

## 目标

1. 降低页面长度和视觉噪音，让用户更直观看到核心信息。
2. 用户核心任务优先级固定为：先「看行情」和「AI 判断」，其次才是「管理持仓」。
3. AI 分析是商业转化核心入口，必须常驻首屏。
4. PC 与移动端访问占比接近，各自都需要清晰的信息架构。
5. 本阶段只做前端 UI 重构，不改后端接口。

## 功能需求

- 首屏保留股票标题、行情摘要、关注入口、AI 分析入口。
- 页面内容按任务域分组展示，避免所有模块纵向平铺。
- PC 端采用六个 Tab：概览、走势、技术、基本面、AI & 资讯、持仓 & 提醒。
- 移动端采用四组入口：概览、走势、分析、持仓；其中「分析」下继续提供技术/基本面/AI 的二级切换。
- URL query `tab` 可恢复当前分类；非法 tab 回退概览。
- 未登录用户访问持仓分类时显示登录引导，不显示空白。
- 原有后端数据源和业务口径保持不变。

## 非功能需求

- 遵守现有主题 token 规范，避免大面积硬编码颜色。
- 不引入后端聚合接口。
- 新增前端信息架构配置需可测试。
- 保持现有 AI、新闻、持仓、信号配置交互能力。

## 验收口径

- `/live-trading/[symbol]` 首屏能看到 AI 分析入口。
- PC 端能在六个 Tab 间切换。
- 移动端显示四组主入口，分析组内可切换技术/基本面/AI。
- `?tab=unknown` 等非法值不会导致页面异常。
- 前端测试通过，构建成功。
