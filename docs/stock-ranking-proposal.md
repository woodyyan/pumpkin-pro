# 股票排行榜功能方案 — 「卧龙AI精选」

> 版本：V1 | 状态：待审核 | 日期：2026-04-15

---

## 一、功能概述

在行情看板页面新增「卧龙AI精选」排行榜模块，从四象限机会区中筛选出 **Top 20** 股票，按 AI 综合评分排序。用户无需分析，直接参考榜单选股。

- **Slogan**: 基于卧龙AI模型每日分析
- **目标用户**: 普通散户（降低选股门槛，「抄作业即用」）
- **展示位置**: 行情看板页面 (`live-trading.js`)，四象限全景图下方独立 Section
- **双市场**: A股排行榜 + 港股排行榜，Tab 切换（复用现有 quadrantExchange 模式）

## 二、排名算法设计

### 2.1 核心思路：机会区内按 Opportunity 降序

你的想法完全合理——四象限的「机会区」已经是一个高质量过滤：

```
机会区定义：opportunity > 70 AND risk < 40
           （高机会 + 低风险）
```

在这个集合内再排 Top 20，本质上是 **"优中选优"**。

### 2.2 排名公式

| 维度 | 权重 | 说明 |
|------|------|------|
| **Opportunity Score** | 主排序键（DESC） | 趋势+资金流+预期修正的综合得分，越高越强 |
| **Risk Score** | 副排序键（ASC） | 当 opportunity 接近时，优先推荐低风险的 |
| **Trend 子分** | 展示用 | 帮助用户理解为什么这只股票排在前面 |

**伪代码 SQL**：
```sql
SELECT * FROM quadrant_scores
WHERE quadrant = '机会'
  AND exchange IN ('SSE', 'SZSE')   -- 或 'HKEX' for HK ranking
ORDER BY opportunity DESC, risk ASC
LIMIT 20
```

### 2.3 为什么这个算法合理？

1. **不引入新的黑箱评分** — 直接复用已有的四象限模型，用户已在看板上理解了这个体系
2. **Opportunity 本身就是加权组合** — Trend(50%) + Flow(30%) + Revision(20%)，已涵盖动量、资金面、基本面
3. **Risk 作为副排序列** — 两只 opportunity 都是 95 分的股票，风险 25 的优于风险 38 的
4. **每日自动刷新** — 四象限每天凌晨重算，排行榜自然跟着更新，不需要独立的计算调度
5. **无额外数据依赖** — 不需要 PE/PB/市值等额外字段，纯四象限分数驱动

### 2.4 边界情况处理

| 场景 | 处理策略 |
|------|----------|
| 机会区不足 20 只 | 有多少只显示多少只，显示实际数量（如 "12 只"） |
| 机会区为 0 只 | 显示空状态：「当前市场暂无明显机会标的」+ 建议关注防御区 |
| 四象限数据未更新（超过 3 天） | 复用现有过期警告逻辑，排行榜区域显示 amber 提示条 |
| 港股首次上线或数据不足 | 港股 Tab 正常显示，数量可能较少，不做特殊处理 |

## 三、数据库设计

### 3.1 方案选择：不建新表，实时查询

**结论：不需要新建 `rankings` 表。**

理由：

1. **查询简单** — `WHERE quadrant='机会' ORDER BY opportunity DESC LIMIT 20`，单表带索引毫秒级返回
2. **数据零冗余** — 排行榜是 `quadrant_scores` 的视图层，不引入一致性问题
3. **自动同步** — 四象限每日重算后排行榜自然更新，无需额外的 ETL 流程
4. **历史可追溯** — 如果未来需要「上周排行 vs 本周排行」，可以从 `quadrant_compute_logs` + 当前数据推算

### 3.2 所需的数据库变更

**无 Schema 变更**。

只需确保现有索引支持该查询模式：

```sql
-- 已有索引（确认即可）
CREATE INDEX idx_quadrant_exchange ON quadrant_scores(exchange);
-- 建议新增复合索引（优化排序查询）
CREATE INDEX idx_quadrant_opp_zone ON quadrant_scores(quadrant, exchange, opportunity DESC, risk ASC);
```

> GORM AutoMigrate 在 migrator.go 中会处理索引创建。


### 5.3 视觉设计规范

| 元素 | 设计细节 |
|------|----------|
| **排名徽章** | Top 3 用 🥇🥈🥉 金/银/铜色；4-10 用深灰圆圈数字；11-20 用浅灰数字 |

| **风险分颜色** | 低分(<30)用绿色（安全），中等(30-50)用琥珀色警告 |
| **进度条** | Opportunity 用渐变绿→蓝，Risk 用渐变绿→红 |
| **空状态** | 友好提示文案 + 引导去四象限图看看防御区选项 |
| **加载态** | 骨架屏（shimmer），与现有 quadrantLoading 风格统一 |

| 操作 | 行为 |
|------|------|
| **点击整行/卡片** | `window.open('/live-trading/{symbol}', '_blank')` 打开详情页 |
| **Tab 切换** | 切换 A股/港股，loading → 渲染新列表，复用 `quadrantExchange` state 或新建独立 `rankingExchange` state |
| **数据刷新** | 跟随行情看板 5s 轮询（可选，也可降频到 60s 因为排行榜变动慢） |
| **显示数量** | 默认 20 只，底部可展开到 50 只（如果机会区超过 20 只）|

## 九、风险与注意事项

| 风险 | 缓解措施 |
|------|----------|
| 机会区股票过少（如 < 5 只） | 正常显示实际数量，配合文案引导 |
| 用户过度依赖排行榜做投资决策 | 页脚加免责声明：「仅供参考，不构成投资建议」 |
| 排行榜变化过大导致用户困惑 | 可考虑加入「连续 N 天上榜」标签（P2） |
| 港股机会区长期偏少 | 港股评分模型本身的问题，非本功能范畴；正常展示即可 |
